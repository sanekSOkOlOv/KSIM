
### Лабораторна робота №**2**. Редагування імітаційних моделей у середовищі NetLogo

### Варіант **5**, модель у середовищі NetLogo: Fire Simple Extension 2

### Внесені зміни у вихідну логіку моделі, за варіантом:

Були додані пожежнки та процедура їх розміщення.

```
breed [firefighters firefighter]

to setup-firefighters
  create-firefighters number-of-firefighters [
    ;; вибрати випадкову порожню клітинку
    move-to one-of patches with [pcolor != red and pcolor != green]
    set color white
    set shape "person"
  ]
end
```

У процедурі setup використовується виклик окремої нової процедури розміщення пожежнків:

```
to setup
  clear-all
  ;; стандартна ініціалізація дерев
  ask patches [
    if (random 100) < density [ set pcolor green ]
    if pxcor = min-pxcor [ set pcolor red ]
  ]
  set initial-trees count patches with [pcolor = green]
  ;;set number-of-firefighters 10
  setup-firefighters ;; виклик розміщення
  reset-ticks
end
```

## Обчислювальні експерименти для Fire Simple Extension 2

### Експеримент 1. Вплив щільності дерев на масштаб пожежі з пожежниками.

**Опис:**  
У цьому експерименті досліджується, як щільність дерев у лісі впливає на масштаб пожежі (з пожжниками). При низькій щільності вогонь може швидко згасати через відсутність сусідніх дерев, тоді як при високій щільності пожежа має більше можливостей для поширення. Очікується, що зі збільшенням щільності зростатиме частка згорілих дерев.

Початкові умови

- **Probability-of-spread:** 60%
- **South-wind-speed:** 25
- **West-wind-speed:** 25
- number-of-firefighters: 1000
- **Змінний параметр:** density ∈ {20, 40, 60, 80, 100}%

| density | Percent borned |
| ------- | -------------- |
| 20      | 2.2%           |
| 40      | 1.7%           |
| 60      | 1.8%           |
| 80      | 23%            |
| 99      | 89.9%          |

### Внесені зміни у вихідну логіку моделі, згідно власного розсуду:

Спочатку я додав глобальну змінну fires-extinguished, яка буде рахувати кількість погашених клітинок.

В setup обнуляємо лічильник:

```
set fires-extinguished 0  ;; обнулити лічильник
```

У місці, де пожежні гасять червоні клітинки, додав:

```
to move-firefighters
  ask firefighters [
    rt random 360
    fd 1

    ;; гасимо всі червоні клітинки поруч
    ask neighbors4 with [pcolor = red] [
      set pcolor black
      set fires-extinguished fires-extinguished + 1
    ]

    ;; якщо сам пожежний стоїть на червоній клітинці — теж гасимо
    if [pcolor] of patch-here = red [
      ask patch-here [
        set pcolor black
        set fires-extinguished fires-extinguished + 1
      ]
    ]
  ]
end
```

У вкладці **Interface** можеш додати **монітор** з назвою fires-extinguished, щоб бачити, як змінюється кількість погашених клітинок у реальному часі.
![[Pasted image 20251119151738.png]]

Далі я ввів два типи дерев з різною стійкістю до вогню ( звичайні та дуже вологі).

```
patches-own [ tree-type ]  ;; тип дерева: "normal" або "resistant"
```

У процедурі `setup`, після того як ти встановлюєш pcolor green, додав

```
if (random 100) < density [
  set pcolor green
  set tree-type one-of ["normal" "resistant"]
]

set initial-trees count patches with [pcolor = green or pcolor = lime]
```

У блоці, де вогонь поширюється на сусідів, додав перевірку типу дерева:

```
if tree-type = "resistant" [
  set probability probability * 0.66 
]
```

Щоб вогонь міг поширюватися і на resistant дерева

```
neighbors4 with [pcolor = green or pcolor = lime]
```
